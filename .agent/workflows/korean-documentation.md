# 한글 문서 작성 가이드

이 워크플로우는 프로젝트 내 모든 문서를 한글로 작성하는 규칙과 가이드라인을 정의합니다.

## 기본 원칙

1. **모든 문서는 한글로 작성**
   - 프로젝트 문서, 주석, README, 가이드 등 모든 문서는 한글로 작성합니다.
   - 코드 내 주석도 한글로 작성하는 것을 권장합니다.

2. **기술 용어 처리**
   - 기술 용어는 필요시 영문을 병기합니다. 예: `상태 관리(State Management)`
   - 일반적으로 통용되는 용어는 영문 그대로 사용 가능합니다. 예: `ViewModel`, `Repository`, `API`

3. **코드 예제**
   - 코드 내 변수명, 함수명은 영문으로 작성합니다.
   - 코드 설명은 한글로 작성합니다.

## 문서 작성 단계

### 1. 새 문서 생성 시
```markdown
# 문서 제목 (한글)

## 개요
문서의 목적과 범위를 한글로 설명합니다.

## 주요 내용
...
```

### 2. 기존 문서 업데이트 시
- 기존 한글 문서를 수정하거나 확장합니다.
- 일관성 있는 용어를 사용합니다.

### 3. 코드 주석 작성 시
```kotlin
/**
 * 사용자 정보를 관리하는 ViewModel
 * 
 * @param repository 사용자 데이터를 가져오는 Repository
 */
class UserViewModel(
    private val repository: UserRepository
) : ViewModel() {
    // 사용자 상태를 관리하는 StateFlow
    private val _userState = MutableStateFlow<UserState>(UserState.Loading)
    val userState: StateFlow<UserState> = _userState.asStateFlow()
}
```

### 4. README 작성 시
- 프로젝트 개요, 설치 방법, 사용법 등을 한글로 작성합니다.
- 필요한 경우 영문 README를 별도로 제공할 수 있습니다.

## 문서 유형별 가이드

### 기술 문서 (docs/)
- 아키텍처, 설계, 구현 가이드 등은 모두 한글로 작성
- 파일명은 한글 또는 영문 모두 가능 (예: `아키텍처-가이드.md` 또는 `architecture-guide.md`)

### 주석 (코드 내)
- 클래스, 함수, 복잡한 로직에 대한 설명은 한글로 작성
- KDoc/JavaDoc 형식을 따르되 내용은 한글로 작성

### 커밋 메시지
- 한글 또는 영문 모두 가능
- 팀 컨벤션에 따라 결정

## 체크리스트

문서 작성 시 다음 사항을 확인하세요:

- [ ] 문서 제목이 명확하고 한글로 작성되었는가?
- [ ] 기술 용어가 일관성 있게 사용되었는가?
- [ ] 코드 예제에 한글 설명이 포함되었는가?
- [ ] 문서 구조가 논리적이고 이해하기 쉬운가?
- [ ] 필요한 경우 영문 용어를 병기했는가?

## 참고 사항

- AI 어시스턴트(Antigravity)는 기본적으로 한글로 응답하도록 설정되어 있습니다.
- 프로젝트별 추가 규칙이 필요한 경우 이 문서를 업데이트하세요.
- 팀원들과 문서화 규칙을 공유하고 일관성을 유지하세요.

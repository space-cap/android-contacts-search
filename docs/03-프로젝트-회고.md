# 03-프로젝트-회고

본 문서는 **ContactsSearch (연락처 초성 검색 앱)** 개발 프로젝트의 전 과정을 되돌아보고, 주요 성과와 배운 점을 정리한 회고 문서입니다.

---

## 1. 프로젝트 개요

- **목적**: 안드로이드 사용자에게 빠르고 직관적인 연락처 검색 및 즐겨찾기 기능을 제공함.
- **기간**: 2026.01.08 ~ 2026.01.09
- **주요 기능**:
    - 안드로이드 시스템 연락처 연동 (ContentProvider)
    - 한글 초성 매칭 알고리즘 기반 실시간 검색
    - Room DB를 활용한 즐겨찾기 영속 저장
    - Modern UI/UX (Jetpack Compose, Material 3)

---

## 2. 주요 구현 상세

### 🛠 기술 스택 및 아키텍처
- **Architecture**: MVVM 패턴을 적용하여 데이터 로직과 UI 로직을 명확히 분리하였습니다.
- **Data Layer**: 시스템 연락처를 읽어오는 `ContactRepository`와 즐겨찾기 정보를 관리하는 Room Database를 통합하여 스트림 기반(`Flow`)으로 데이터를 제공합니다.
- **UI Layer**: Jetpack Compose를 사용하여 선언형 UI를 구축하였으며, `StateFlow`와 `combine` 연산자를 활용해 검색어 변화와 즐겨찾기 상태를 실시간으로 반응하도록 구현하였습니다.

### 🔍 핵심 로직: 초성 검색
- 한국어의 특성을 고려하여, 사용자가 "ㄱㅎㄷ"과 같이 초성만 입력해도 "김현도"를 찾을 수 있는 매칭 알고리즘(`KoreanUtils`)을 구현하였습니다.
- 정규표현식과 유니코드 범위를 활용하여 정확하고 빠른 필터링 성능을 확보하였습니다.

---

## 3. 성공적인 점

- **기능 완성도**: 계획했던 핵심 기능(초성 검색, 즐겨찾기, 권한 처리)을 모두 성공적으로 구현하였습니다.
- **데이터 통합**: 시스템 데이터(연락처)와 로컬 DB 데이터(즐겨찾기)를 하나의 UI 모델(`Contact`)로 매끄럽게 결합하여 일관된 사용자 경험을 제공했습니다.
- **코드 품질**: 수동 DI(Dependency Injection) 구조를 도입하여 확장성과 유지보수성을 고려한 설계를 적용했습니다.

---

## 4. 학습 및 도전 과제

### ❗ 도전 과제: 데이터 결합 및 동기화
- 시스템 연락처 데이터는 실시간으로 변경될 수 있고, 즐겨찾기 상태는 앱 내 로컬 DB에 존재합니다. 이 두 소스에서 오는 데이터를 효율적으로 결합하여 UI에 전달하는 과정이 도전적이었습니다.
- **해결**: `combine` 함수를 사용하여 두 `Flow`를 하나로 합치고, 데이터가 변경될 때마다 UI가 자동으로 갱신되도록 설계하여 문제를 해결했습니다.

### 💡 학습한 점
- Jetpack Compose에서의 상태 관리(`StateFlow`)와 비동기 데이터 처리(`Coroutines`)의 깊은 연계 과정을 실무적으로 익힐 수 있었습니다.
- 권한 처리(Permissions)를 Compose 라이프사이클 내에서 자연스럽게 통합하는 방식을 실습했습니다.

---

## 5. 향후 개선 계획

- **그룹화 및 정렬**: 연락처를 가나다 순으로 그룹화하거나, 최근 통화 기록과의 연동 기능을 고려할 수 있습니다.
- **성능 최적화**: 연락처 개수가 매우 많을 때를 대비한 페이징 처리(Paging 3) 도입을 검토할 수 있습니다.
- **테마 확장**: 다크 모드 및 사용자 커스텀 컬러 테마 지원.

# 02-즐겨찾기-기능-구현-작업-계획서

## 프로젝트 개요

### 목적
연락처 목록에서 사용자가 특정 연락처를 즐겨찾기로 등록하고, 이를 로컬 데이터베이스에 저장하여 영구적으로 관리할 수 있는 기능을 추가합니다.

### 주요 기능
1. **즐겨찾기 등록/해제**: 연락처 리스트의 각 아이템에서 즐겨찾기를 토글할 수 있습니다.
2. **데이터 영속성**: Room Database를 사용하여 즐겨찾기 상태를 저장하므로 앱 재시작 시에도 유지됩니다.
3. **즐겨찾기 표시**: 즐겨찾기가 등록된 연락처는 별 모양 아이콘으로 시각적 구분을 제공합니다.

---

## 구현 계획

### 1. 기술 스택
- **Database**: Room Persistence Library
- **Architecture**: MVVM 패턴 유지
- **UI**: Jetpack Compose (Material3 Icons)

### 2. 데이터 모델 변경
- `Contact.kt` 모델에 `isFavorite: Boolean` 필드 추가
- `FavoriteContact.kt` (Room Entity) 추가: 시스템 연락처 ID(`contactId`)만 저장

### 3. 주요 구현 단계

#### Phase 1: Room DB 설정
- Room 의존성 추가 (`build.gradle.kts`)
- Entity, Dao, Database 클래스 구현

#### Phase 2: 데이터 레이어 연동
- `ContactRepository`에서 시스템 연락처 데이터와 Room DB 즐겨찾기 데이터를 결합하는 로직 구현

#### Phase 3: UI 및 ViewModel 업데이트
- `ContactViewModel`에 즐겨찾기 토글 비즈니스 로직 추가
- `ContactItem` 컴포저블에 즐겨찾기 상태 표시 및 클릭 이벤트 추가

---

## 참고 사항
- 시스템 연락처 데이터 자체를 수정하는 것이 아니라, 사용자가 앱 내에서 지정한 "즐겨찾기" 정보만을 별도 DB에 관리합니다.
- 향후 즐겨찾기한 연락처만 상단에 모아보기 기능을 추가할 수 있습니다.
